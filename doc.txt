دليلك الفني وتوثيق واجهة برمجة التطبيقات (API Documentation)

المشروع مبني بهيكل مخصص ليعمل بكفاءة على بيئات الحوسبة السحابية وبدون خادم (Serverless) على منصة Vercel مع ربطه بقاعدة بيانات PostgreSQL عبر متقاطع البيانات Prisma.
1. إعداد البيئة والتشغيل المحلي (Setup & Installation)
المتطلبات المسبقة

    تثبيت Node.js (الإصدار 18 أو أحدث).

    حساب على Neon Database للحصول على رابط اتصال PostgreSQL.

    حساب على Cloudinary لرفع وإدارة صور المنتجات والتصنيفات.

خطوات التشغيل

    قم بفك ضغط ملفات المشروع داخل مجلد خاص.

    افتح وحدة التحكم (Terminal) داخل مجلد المشروع وقم بتثبيت الحزم:
    Bash

    npm install

    قم بإنشاء ملف .env في المجلد الرئيسي بناءً على نموذج .env.example واملأ البيانات المطلوبة.

    قم بإنشاء جداول قاعدة البيانات وتوليد عميل بريسما (Prisma Client) عبر الأمر التالي:
    Bash

    npx prisma db push

    لتشغيل المشروع محلياً في بيئة التطوير:
    Bash

    npm run dev

2. الهيكل العام للمسارات (Route Architecture)

تتطلب جميع المسارات المحمية إرسال توكن التحقق في عنوان الطلب (Headers) بالشكل التالي:
Authorization: Bearer <Your_JWT_Token>
مسارات الحماية والتوثيق (Authentication API)

    جميع المدخلات ترسل عبر صيغة JSON في الـ Body.

الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
تسجيل حساب جديد	/api/auth/register	POST	للجميع	يتطلب: name, email, password
تسجيل الدخول	/api/auth/login	POST	للجميع	يتطلب: email, password
جلب بيانات الحساب	/api/auth/profile	GET	للمستخدم المسجل	يعيد بيانات المستخدم صاحب التوكن
مسارات المنتجات (Products API)

    تدعم دالة جلب كل المنتجات ميزات برمجية متقدمة عبر الـ Query Parameters:

        الصفحات: ?page=1&limit=10

        البحث: ?search=اسم_المنتج

        التصنيف: ?categoryId=معرف_التصنيف

        الترتيب السعري: ?sort=price_asc أو ?sort=price_desc

        نطاق السعر: ?minPrice=50&maxPrice=500

الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
جلب جميع المنتجات	/api/products	GET	للجميع	يدعم الفلترة والتصفح
جلب منتج محدد	/api/products/:id	GET	للجميع	يتضمن بيانات التصنيف والمراجعات
إنشاء منتج جديد	/api/products	POST	الأدمن فقط	يرسل كـ multipart/form-data مع حقل images
تحديث منتج	/api/products/:id	PUT	الأدمن فقط	يرسل التحديثات كـ multipart/form-data
حذف منتج	/api/products/:id	DELETE	الأدمن فقط	يحذف المنتج نهائياً من الجداول
مسارات التصنيفات (Categories API)
الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
جلب كل التصنيفات	/api/categories	GET	للجميع	يعيد القائمة كاملة بالصور
جلب تصنيف محدد	/api/categories/:id	GET	للجميع	جلب تفاصيل تصنيف واحد
إنشاء تصنيف	/api/categories	POST	الأدمن فقط	يتطلب name وحقل image كملف
تحديث تصنيف	/api/categories/:id	PUT	الأدمن فقط	لتعديل الاسم أو الصورة
حذف تصنيف	/api/categories/:id	DELETE	الأدمن فقط	حذف تصنيف (يحذف المنتجات التابعة له تلقائياً)
مسارات عربة التسوق (Cart API)
الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
عرض محتويات العربة	/api/cart	GET	للمستخدم المسجل	تجلب العناصر مع تفاصيل المنتج
إضافة منتج للعربة	/api/cart	POST	للمستخدم المسجل	يرسل productId و quantity
تحديث كمية المنتج	/api/cart/:id	PUT	للمستخدم المسجل	يرسل حقل quantity الجديد في الـ Body
حذف عنصر من العربة	/api/cart/:id	DELETE	للمستخدم المسجل	يحذف العنصر المحدد من العربة
مسارات الطلبات (Orders API)
الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
إنشاء طلب جديد	/api/orders	POST	للمستخدم المسجل	يتطلب shippingAddress و paymentMethod. يقوم بتفريغ العربة وتعديل المخزون تلقائياً
جلب الطلبات	/api/orders	GET	مستخدم / أدمن	للمستخدم: يعيد طلباته فقط. للأدمن: يعيد كافة طلبات المتجر
تحديث حالة الطلب	/api/orders/:id/status	PUT	الأدمن فقط	يتطلب حقل status في الـ Body بقيم مثل: PROCESSING, SHIPPED, DELIVERED, CANCELLED
مسارات قائمة الرغبات والمراجعات (Wishlist & Reviews API)
الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
جلب قائمة الرغبات	/api/wishlist	GET	للمستخدم المسجل	يعيد المنتجات المفضلة للمستخدم
إضافة لقائمة الرغبات	/api/wishlist	POST	للمستخدم المسجل	يتطلب productId
حذف من الرغبات	/api/wishlist/:id	DELETE	للمستخدم المسجل	يمرر معرف عنصر قائمة الرغبات المراد حذفه
جلب تقييمات منتج	/api/reviews/:productId	GET	للجميع	يعيد قائمة المراجعات والتعليقات للمنتج
إضافة تقييم منتج	/api/reviews	POST	للمستخدم المسجل	يتطلب productId, rating (من 1 لـ 5), و comment
مسارات الكوبونات ولوحة التحكم (Coupons & Admin Dashboard API)
الوظيفة	المسار (Route)	الطريقة (Method)	الصلاحية (Access)	الملاحظات / المدخلات
إنشاء كوبون خصم	/api/coupons	POST	الأدمن فقط	يتطلب code, discount كنسبة مئوية، و expiryDate
التحقق من الكوبون	/api/coupons/validate	POST	للمستخدم المسجل	يتطلب حقل code. يعيد نسبة الخصم إذا كان فعالاً وغير منتهي الصلاحية
إحصائيات لوحة التحكم	/api/dashboard/stats	GET	الأدمن فقط	يعيد أعداد المستخدمين، المنتجات، المبيعات الإجمالية، وآخر 5 طلبات
3. التعامل مع الأخطاء والحماية (Error Handling & Security)

    محدد الطلبات (Rate Limiting): تم ضبط النظام لمنع الهجمات البرمجية بحيث يسمح بـ 100 طلب كحد أقصى لكل 15 دقيقة لكل عنوان IP على مسارات الـ API.

    الحماية (Helmet): يقوم بروتوكول helmet بإعداد ترويسات HTTP الآمنة تلقائياً لحماية التطبيق من الثغرات الشائعة مثل XSS
